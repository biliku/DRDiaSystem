# 治疗方案推荐与医患交流模块功能设计文档

## 一、治疗方案推荐模块

### 1.1 功能概述
基于AI诊断结果和患者病情信息，为医生提供智能治疗方案推荐，医生可在此基础上制定个性化治疗方案。

### 1.2 核心功能

#### 1.2.1 AI智能推荐（医生端）
- **基于诊断报告推荐**：
  - 根据DR分级（0-4级）推荐对应治疗方案
  - 根据病灶类型（微血管瘤、出血、硬性渗出、软性渗出、新生血管等）推荐针对性治疗
  - 结合患者糖尿病类型、病程、血糖控制情况推荐
- **推荐内容包含**：
  - 药物治疗方案（药物名称、用法用量、疗程）
  - 复查计划（复查时间、检查项目）
  - 生活方式建议（饮食、运动、血糖监测）
  - 注意事项和风险提示

#### 1.2.2 治疗方案管理（医生端）
- **创建治疗方案**：
  - 关联病例记录（CaseRecord）
  - 关联诊断报告（DiagnosisReport）
  - 选择推荐方案或自定义方案
  - 设置方案状态（草稿/已确认/已执行/已结束）
- **编辑治疗方案**：
  - 修改药物、复查计划、建议等
  - 记录方案变更历史
- **方案确认与下发**：
  - 医生确认后，方案自动通知患者
  - 患者可查看治疗方案详情

#### 1.2.3 治疗方案查看（患者端）
- 查看当前治疗方案
- 查看历史治疗方案
- 查看方案执行进度
- 记录方案执行反馈

# TreatmentPlan 治疗方案模型
class TreatmentPlan(models.Model):
    STATUS_CHOICES = (
        ('draft', '草稿'),
        ('confirmed', '已确认'),
        ('active', '执行中'),
        ('completed', '已完成'),
        ('cancelled', '已取消'),
    )
    
    case = models.ForeignKey(CaseRecord, on_delete=models.CASCADE, related_name='treatment_plans')
    related_report = models.ForeignKey(DiagnosisReport, on_delete=models.SET_NULL, null=True, blank=True)
    
    # 方案基本信息
    plan_number = models.CharField(max_length=100, unique=True)  # 方案编号
    title = models.CharField(max_length=200)  # 方案标题
    status = models.CharField(max_length=20, choices=STATUS_CHOICES, default='draft')
    
    # 治疗方案内容（JSON格式）
    medications = models.JSONField(default=list)  # 药物列表
    # [{"name": "药物名称", "dosage": "用法用量", "duration": "疗程", "notes": "备注"}]
    
    follow_up_plan = models.JSONField(default=dict)  # 复查计划
    # {"next_date": "2024-01-15", "check_items": ["眼底检查", "血糖检测"], "interval_days": 90}
    
    lifestyle_advice = models.TextField(blank=True)  # 生活方式建议
    precautions = models.TextField(blank=True)  # 注意事项
    
    # AI推荐信息
    is_ai_recommended = models.BooleanField(default=False)  # 是否AI推荐
    ai_recommendation_score = models.FloatField(null=True, blank=True)  # 推荐置信度
    
    # 医生信息
    created_by = models.ForeignKey(User, on_delete=models.SET_NULL, null=True, related_name='created_treatment_plans')
    confirmed_by = models.ForeignKey(User, on_delete=models.SET_NULL, null=True, blank=True, related_name='confirmed_treatment_plans')
    confirmed_at = models.DateTimeField(null=True, blank=True)
    
    # 时间戳
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
    start_date = models.DateField(null=True, blank=True)  # 开始执行日期
    end_date = models.DateField(null=True, blank=True)  # 预计结束日期

# TreatmentPlanExecution 方案执行记录
class TreatmentPlanExecution(models.Model):
    plan = models.ForeignKey(TreatmentPlan, on_delete=models.CASCADE, related_name='executions')
    execution_date = models.DateField()  # 执行日期
    medication_taken = models.JSONField(default=dict)  # 用药记录
    follow_up_completed = models.BooleanField(default=False)  # 是否完成复查
    patient_feedback = models.TextField(blank=True)  # 患者反馈
    doctor_notes = models.TextField(blank=True)  # 医生备注
    created_at = models.DateTimeField(auto_now_add=True)

# TreatmentPlanTemplate 治疗方案模板（用于AI推荐）
class TreatmentPlanTemplate(models.Model):
    """治疗方案模板，用于AI推荐"""
    dr_grade = models.IntegerField()  # DR分级 0-4
    lesion_types = models.JSONField(default=list)  # 适用病灶类型
    diabetes_type = models.CharField(max_length=20, blank=True)  # 适用糖尿病类型
    
    # 模板内容
    medications = models.JSONField(default=list)
    follow_up_plan = models.JSONField(default=dict)
    lifestyle_advice = models.TextField()
    precautions = models.TextField()
    
    priority = models.IntegerField(default=0)  # 优先级
    is_active = models.BooleanField(default=True)
    created_at = models.DateTimeField(auto_now_add=True)
```

## 二、医患交流模块

### 2.1 功能概述
为医生和患者提供实时沟通平台，支持文字消息、图片、文件传输，可关联病例和报告进行讨论。

### 2.2 核心功能

#### 2.2.1 消息交流
- **文字消息**：支持富文本编辑、表情
- **图片消息**：上传眼底照片、检查报告等
- **文件消息**：上传PDF报告、检查单等
- **消息状态**：已发送、已送达、已读

#### 2.2.2 会话管理
- **会话列表**：
  - 医生端：显示所有患者会话，按最后消息时间排序
  - 患者端：显示与医生的会话
- **会话详情**：
  - 显示患者基本信息
  - 关联的病例和报告
  - 消息历史记录
  - 支持搜索消息

#### 2.2.3 消息关联
- 可关联到具体病例（CaseRecord）
- 可关联到诊断报告（DiagnosisReport）
- 可关联到治疗方案（TreatmentPlan）
- 关联后可在对应页面查看相关消息

#### 2.2.4 通知提醒
- 新消息实时通知（WebSocket或轮询）
- 未读消息数量提示
- 重要消息标记（医生标记）

#### 2.2.5 快捷回复（医生端）
- 常用回复模板
- 快速引用诊断报告
- 快速引用治疗方案

# Conversation 会话模型
class Conversation(models.Model):
    """医患会话"""
    patient = models.ForeignKey(User, on_delete=models.CASCADE, related_name='patient_conversations')
    doctor = models.ForeignKey(User, on_delete=models.CASCADE, related_name='doctor_conversations')
    
    # 关联信息
    related_case = models.ForeignKey(CaseRecord, on_delete=models.SET_NULL, null=True, blank=True, related_name='conversations')
    
    # 会话状态
    is_active = models.BooleanField(default=True)  # 是否活跃
    patient_unread_count = models.IntegerField(default=0)  # 患者未读数
    doctor_unread_count = models.IntegerField(default=0)  # 医生未读数
    
    # 最后消息
    last_message = models.TextField(blank=True)
    last_message_at = models.DateTimeField(null=True, blank=True)
    last_message_by = models.ForeignKey(User, on_delete=models.SET_NULL, null=True, blank=True, related_name='last_message_conversations')
    
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
    
    class Meta:
        unique_together = ['patient', 'doctor']
        ordering = ['-last_message_at']

# Message 消息模型
class Message(models.Model):
    MESSAGE_TYPE_CHOICES = (
        ('text', '文字'),
        ('image', '图片'),
        ('file', '文件'),
        ('system', '系统消息'),
    )
    
    conversation = models.ForeignKey(Conversation, on_delete=models.CASCADE, related_name='messages')
    sender = models.ForeignKey(User, on_delete=models.CASCADE, related_name='sent_messages')
    
    # 消息内容
    message_type = models.CharField(max_length=20, choices=MESSAGE_TYPE_CHOICES, default='text')
    content = models.TextField()  # 文字内容或文件路径
    file_url = models.CharField(max_length=500, blank=True)  # 文件URL（图片/文件）
    file_name = models.CharField(max_length=200, blank=True)  # 文件名
    
    # 关联信息
    related_report = models.ForeignKey(DiagnosisReport, on_delete=models.SET_NULL, null=True, blank=True, related_name='messages')
    related_treatment_plan = models.ForeignKey(TreatmentPlan, on_delete=models.SET_NULL, null=True, blank=True, related_name='messages')
    
    # 消息状态
    is_read = models.BooleanField(default=False)
    read_at = models.DateTimeField(null=True, blank=True)
    is_important = models.BooleanField(default=False)  # 重要消息标记
    
    created_at = models.DateTimeField(auto_now_add=True)
    
    class Meta:
        ordering = ['created_at']

# MessageTemplate 消息模板（医生快捷回复）
class MessageTemplate(models.Model):
    """消息模板"""
    doctor = models.ForeignKey(User, on_delete=models.CASCADE, related_name='message_templates')
    title = models.CharField(max_length=100)  # 模板标题
    content = models.TextField()  # 模板内容
    category = models.CharField(max_length=50, blank=True)  # 分类（如：诊断说明、复查提醒等）
    is_public = models.BooleanField(default=False)  # 是否公开（所有医生可用）
    usage_count = models.IntegerField(default=0)  # 使用次数
    created_at = models.DateTimeField(auto_now_add=True)
```

## 四、业务流程

### 4.1 治疗方案推荐流程

```
1. 医生查看病例详情
   ↓
2. 点击"生成治疗方案推荐"
   ↓
3. 系统基于诊断报告和患者信息生成推荐
   ↓
4. 医生查看推荐方案（可查看多个方案对比）
   ↓
5. 医生选择或编辑方案
   ↓
6. 医生确认方案
   ↓
7. 系统通知患者
   ↓
8. 患者查看并执行方案
   ↓
9. 患者记录执行情况
   ↓
10. 医生查看执行反馈，调整方案
```

### 4.2 医患交流流程

```
1. 患者发起咨询（或医生主动联系）
   ↓
2. 创建/进入会话
   ↓
3. 发送消息（可关联报告/方案）
   ↓
4. 对方收到通知
   ↓
5. 查看并回复消息
   ↓
6. 持续交流
   ↓
7. 医生可标记重要消息
   ↓
8. 会话可关联到病例记录
```

## 五、技术实现要点

### 5.1 AI推荐算法
- 基于规则引擎：DR分级 + 病灶类型 → 治疗方案模板匹配
- 可扩展：未来可集成机器学习模型

### 5.2 实时通信
- 方案1：WebSocket（Django Channels）
- 方案2：轮询（简单实现，适合初期）

### 5.3 文件存储
- 消息中的图片/文件存储在media/conversations/
- 支持图片预览、文件下载

### 5.4 权限控制
- 患者只能查看自己的方案和消息
- 医生可查看所有相关患者的信息
- 管理员拥有全部权限
